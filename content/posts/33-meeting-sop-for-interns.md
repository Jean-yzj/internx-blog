# 開會不再緊張：實習生的會議生存 SOP

「會議裡都是資深的人，我該不該發言？」「主管要我『一起來開會』，但沒交代我要做什麼。」「會議結束我筆記寫了一堆，回家完全看不懂。」對實習生來說，會議是最容易產生焦慮的場景之一——既怕說錯話、又怕坐在角落像是隱形人。但其實只要建立一套會前、會中、會後的 SOP，會議就會從「壓力源」變成「最快速展示專業度的舞台」。本篇文章帶你拆解每個階段該做什麼，附上會後 follow up email 範本和「要不要錄音」的實戰建議。

## 一、會前 30 分鐘：把會議變成可預測的場景

很多實習生緊張的根源，是「不知道會發生什麼」。會前 30 分鐘的準備，可以把不確定降到最低。

### 1. 確認你被邀請的原因
打開會議邀請，看看：
- 主旨是什麼
- 與會者有誰、層級多高
- 你被列為「必要出席」還是「選擇性出席」

如果不清楚為什麼找你來，私訊邀請人：「想跟你確認一下，你希望我這次會議貢獻什麼？是來學習、做筆記，還是會被點名分享 X 進度？」這一句話會大幅降低你的不確定感。

### 2. 預讀所有相關資料
- 會議邀請有附文件 → 全部讀完
- 沒附文件 → 自己去翻 Notion / Google Drive / Slack 找上次同主題的紀錄
- 找不到 → 提早 15 分鐘問同事「上次這個議題討論到哪裡」

### 3. 準備兩個問題
就算你只是來旁聽，也要準備兩個問題放在口袋裡。發言時機到時，比起硬擠想法，準備好的問題會讓你看起來有備而來。

### 4. 確認硬體
- 線上會議：耳機、攝影機、網路、會議連結提早 5 分鐘測試
- 實體會議：印好需要的資料、帶筆記本、確認會議室位置

## 二、會中：發言時機與記筆記的方法

### 1. 第一次參加新團隊的會議：先觀察、再參與
第一次或第二次會議，建議 80% 時間聽、20% 時間問。聽什麼？
- 誰是真正的決策者（不一定是頭銜最大的）
- 誰會打斷誰、誰會被打斷（決定發言順序的潛規則）
- 哪些議題會被快速通過、哪些會被反覆討論
- 會議的節奏：先 brainstorming 還是先決策？

### 2. 何時發言才不會踩雷
**安全的發言時機**：
- 主席主動點名你：「這個你怎麼看？」
- 議題剛好跟你正在做的任務有關
- 別人問了一個問題，剛好你查過

**有風險的發言時機**：
- 兩位資深員工在爭論時，不要插嘴
- 你有完全不同的想法但還沒驗證 → 會後私下找主管討論
- 你只是想刷存在感

### 3. 發言的萬用句型
不知道怎麼開口？這幾句先背下來：

- 「我這邊有一個小觀察可以分享，不確定是否相關……」（低風險破冰）
- 「我想確認一下我的理解，剛剛的結論是 A 對嗎？」（確認 + 展現有在聽）
- 「我有一個問題可能很基本，但 X 是什麼意思？」（坦承 + 不丟臉）
- 「如果這個任務由我接，我會需要 Y 資源，可以嗎？」（主動承接）

### 4. 做筆記的方法：用 4 欄結構

不要只是逐字記下大家說的話。試試這個四欄結構：

| 欄位 | 內容 |
|------|------|
| 決策（D） | 會議中拍板的結論 |
| 行動（A） | 待辦事項，註明負責人與時間 |
| 問題（Q） | 還沒回答的問題 |
| 觀察（O） | 你個人的觀察、學習、想法 |

筆記的目的不是記下「全部」，而是記下「會後還用得到」的東西。

## 三、會中 NG 行為：讓主管搖頭的 8 個動作

| NG 行為 | 為什麼不好 |
|---------|-----------|
| 整場會議滑手機 | 比缺席還糟，會被認為不尊重 |
| 一直打斷別人 | 即使你的觀點是對的，也會被歸類為「不會看時機」 |
| 開鏡頭背景一片亂 | 不專業，建議用虛擬背景或對著乾淨牆面 |
| 線上會議全程靜音不發言 | 看起來像沒進來 |
| 不做筆記只用大腦記 | 表示你沒打算 follow up |
| 提問前不先說「我的問題是」 | 容易被誤認為閒聊 |
| 會議結束後立刻消失 | 失去快速問主管問題的好機會 |
| 在會議後才知道「啊我應該要做某件事」 | 表示你會中沒有確認 action item |

## 四、會後 30 分鐘：你跟其他人拉開差距的關鍵時段

90% 的實習生會議結束就走人，剩下 10% 會做這件事——所以會被主管記住。

### 1. 整理筆記成 takeaway
把你的四欄筆記濃縮成三點 takeaway，傳給主管或寄給自己當 reference。

### 2. 寫一封 follow up email（範本）

```
主旨：【會議紀錄】OO 月 OO 日 X 專案週會 takeaway

各位好，

剛剛會議中討論到的重點整理如下：

【決策】
1. 下週起 A 功能由林同事負責，6/15 前完成。
2. B 提案延後到下個月討論，由我先彙整資料。

【待辦】
- 王主管：6/10 前確認預算
- 我：6/12 前完成競品分析（會放在共用資料夾）
- 設計組：6/13 前提供 mockup

【待釐清】
- C 議題尚未拍板，是否下次會議再討論？

如果有任何補充或調整，請隨時告訴我。

謝謝
OOO（實習生）
```

這封 email 的價值不是給別人看，而是讓你在團隊中被歸類為「會 own 會議結論的人」。即使你只是實習生，主動寫一份會議紀錄會讓主管立刻記住你。

### 3. 自己 follow up 的事項，當天就動
會議中你承諾要做的事情，當天結束前先動 30%。即使只是先建一個資料夾、開一個草稿、傳訊息給合作對象——「會議當天有動作」會讓承諾變成可信度。

### 4. 找主管要 5 分鐘 debrief
會議結束後，找主管問：「我可以跟你聊 5 分鐘嗎？我想確認我對剛剛會議的理解。」這個動作會讓你比同期實習生更快進入狀況。

## 五、要不要錄音？實戰判斷指南

### 1. 內部會議
- 多數公司允許，但**一定要先口頭詢問**：「我可以錄音給自己事後複習嗎？」
- 不要錄影，避免儀容、肢體被記錄。
- 錄音檔不上傳雲端、不分享給任何人。

### 2. 客戶會議或敏感議題
- 多數情況下不適合錄音，即使技術上可以。
- 如果一定要紀錄，建議用「逐字筆記」配「會後立刻整理」。

### 3. 線上會議的內建錄影功能
- 如果是主持人開的，可以使用，但仍要先告知與會者。
- 如果你是參與者，不要私下啟動錄影功能。

### 4. 替代方案
比起錄音，更建議：
- 認真做 4 欄筆記
- 會後立刻寫 follow up email
- 找主管 debrief

這三樣加起來的效果，比錄音再回去聽更好。

## 六、不同類型會議的應對策略

### 1. 部門站立會議（standup）
- 每個人 1–2 分鐘輪流講進度
- 你要準備：昨天做了什麼、今天要做什麼、卡關
- 千萬不要超時

### 2. 跨部門協作會議
- 重點觀察各部門的 KPI 衝突點
- 你的角色：做筆記、會後寄 follow up
- 不要在會議中表態「哪邊比較對」

### 3. 客戶會議
- 你的角色多半是學徒，**少講話、多紀錄**
- 會後一定要交一份正式紀錄給主管確認再寄出
- 客戶名稱、職稱、要求一個字都不能寫錯

### 4. 1-on-1
- 帶著議題去，不要等主管問
- 結尾主動 recap：「所以這次的結論是 A、B、C，對嗎？」
- 會後寄 takeaway

### 5. Brainstorming（腦力激盪）
- 數量重於品質，多丟想法
- 不要批評別人的想法
- 會後 follow up「我們選了哪個方向」

## 七、FAQ：實習生最常問的 5 個會議問題

**Q1：我什麼都不懂，可以全程不發言嗎？**
可以，但會後要主動找主管問問題。完全不發言的代價是「你的存在被遺忘」。

**Q2：會議裡有人講話我聽不懂專業術語，怎麼辦？**
當下記下術語，會後立刻 Google 或問同事。不要中途打斷會議流程。

**Q3：主管在會議裡批評我，我該當場回應嗎？**
不要當場辯解。回一句：「我了解了，會後我想跟你進一步討論細節。」會議結束後再單獨找他。

**Q4：我發現會議中討論的方向有錯，我該說嗎？**
看你的把握度。70% 以上 → 委婉提出：「我有一個小疑問，X 這個假設我們有驗證過嗎？」70% 以下 → 會後私下找主管討論。

**Q5：會議拖很久，我下個會議要遲到了，怎麼辦？**
提前 5 分鐘私訊主管：「我下個會議快到了，方便先離開嗎？」不要默默消失。

## 八、結語：會議是實習期最高 ROI 的學習場景

對實習生來說，會議不只是工作流程，更是觀察主管思維、團隊權力結構、產業 know-how 的免費課堂。會前準備、會中筆記、會後 follow up 三個動作做到位，你很快就能從「角落實習生」轉變為「主管願意主動帶著去客戶會議的實習生」。

**請記得三個關鍵原則**：
1. **會前 30 分鐘的準備，決定會中 90 分鐘的價值**：不要只是「赴約」，要「準備上場」。
2. **筆記要記下決策與行動，不要記下逐字稿**：人會忘記細節，但會記得結論。
3. **會後一封 follow up email，是實習生最強的個人品牌建立**：少有人做，做了就被記住。

**行動建議**：到實習通搜尋你公司前輩寫的心得，特別注意他們對「會議文化」的描述——有些公司高頻會議、有些公司用文件取代會議。提前理解你即將進入的會議節奏，可以幫你少走兩週的彎路。
