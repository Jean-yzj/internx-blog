# Side Project 的力量：用業餘時間累積實戰經驗

「我沒有實習機會，怎麼累積經驗？」——這是大學生常見的焦慮。但其實，**Side Project（業餘專案）** 是過去十年最改變學生履歷的趨勢之一。許多頂尖工程師、設計師、PM 都是從一個小 side project 開始進入職場。本篇將帶你看懂 side project 的真正威力與實戰做法。

> 註：以下提到的平台、工具、社群、訂閱／互動率數據都會隨時間變動，**請以該平台當期官方頁面與最新業界報告為準**。

## 一、什麼是 Side Project？

任何「不是課業也不是工作」的個人專案，都算 side project。

### 例子

- 自己寫一個 App、網站、Bot。
- 經營一個 IG / Threads / Newsletter / Podcast。
- 為一個品牌做免費的設計提案。
- 解一份公開資料集（如政府資料開放平台 data.gov.tw、Kaggle）做分析。
- 寫一篇深度的產業研究。
- 拍一個 YouTube 頻道。
- 開發一個 Chrome Extension。
- 做一個 Discord / LINE Bot（台灣大學生最常見的入門題：LINE Notify 提醒、選課查詢 Bot）。
- 用 no-code 工具（Bubble、Webflow、Glide、Notion + Make/Zapier）做一個小工具。

### Side Project 與其他活動的差異

| 活動 | 主導者 | 時間 | 成果歸屬 |
| --- | --- | --- | --- |
| 課業 | 老師 | 學期 | 個人 |
| 實習 | 公司 | 數月 | 公司 |
| 社團 | 集體 | 學期 | 社團 |
| Side Project | **你自己** | 任意 | **你** |

關鍵在於「**你是發起者、決策者、執行者、收割者**」。

## 二、Side Project 為什麼比實習還有用？

> 註：以下不是說「side project 一定贏實習」——而是說「兩者展現的能力不同，side project 補實習補不到的部分」。最好的組合是「實習＋side project」一起做。

### 1. 你是專案的擁有者

實習通常是執行別人定義好的任務，但 side project 是你自己定義問題、自己決策、自己交付。**這展現的是「主導力」**。

### 2. 它是你的長期資產

實習結束後，工作成果可能屬於公司（依各公司合約），但 side project 永遠是你的。

具體價值：

- 數年後你還能拿出來談。
- 可以放進個人網站、LinkedIn、CakeResume、Newsletter。
- 跨公司轉職時依然有效。

### 3. 它能跨越科系與背景

HR 不一定在乎你是哪個系，他更在乎你做出了什麼。

- 中文系做出有真實使用者的 App，可能比資工系只有課程作業更有說服力。
- 哲學系經營一個穩定更新的 Newsletter，可能比商管系只有課業更能展現續航力。

（以上為示意對比，實際錄取結果仍依公司、職務、整體履歷而定。）

### 4. 它是「最低成本」的差異化

其他候選人都有實習，但只有你有「自己經營一年的 Newsletter」或「自己上架 Chrome 商店、有真實使用者的小工具」。

### 5. 它能讓你「真的會做事」

實習中你可能只做了某個小區塊。Side Project 你必須**從 0 到 1 完整執行**：

- 想 idea。
- 找用戶。
- 設計。
- 開發。
- 上線。
- 行銷。
- 維護。

這個全流程經驗，是大多數實習給不了的（除非你做的是新創早期實習）。

### 6. 它能驗證你的職涯方向

想做 PM？做一個 App 看你能不能享受。
想做設計？做設計案例看你能不能持續。
想做寫作？經營 Newsletter 看你能不能輸出。

## 三、好的 Side Project 應有的 5 個特質

### 1. 真實解決一個問題

不要只是「我想做」，要有「為誰、為什麼」。Indie hacker 圈常說的一句話是「Scratch your own itch（先解決你自己的痛）」——這是大學生最容易切入的角度。

### 2. 可被驗證

有用戶、有數據、有回饋，能證明你的成果。

### 3. 能被簡單描述

1 句話講清楚（例如：「我做了一個幫大學生找實習的工具」）。

### 4. 可被分享

做出來後一定要公開，不要做完就丟到資料夾。**沒分享 = 沒做過**。

### 5. 能持續迭代

最好能維持 3–6 個月，看到成長曲線。

## 四、不同興趣的 side project 範例

> 以下範例只是起點，不是「必做清單」。挑一個跟你自己生活有關的題目最容易撐下去。

### 想當 PM？

**簡單版（4 週）**：

- 為一款常用 App（如 Dcard、實習通、台鐵訂票、台灣高鐵 T-EX、健保快易通）寫「3 個改進建議」並發在 Medium / Threads / vocus。

**中階版（2–3 個月）**：

- 用 Notion 模擬一份產品 Roadmap。
- 找 5 位用戶做訪談並輸出洞察報告。
- 為某產業（如台灣外送、二手書、寵物市場）寫一份完整的市場分析。

**高階版（6 個月+）**：

- 自己做一個小型工具，從用戶研究、設計、實作（可考慮 no-code：Bubble、Glide、Softr、Webflow）、上線到推廣。

### 想當設計師？

**簡單版**：完成 Daily UI 100 天挑戰（Figma Community 上有大量 100 Daily UI Challenge 模板可起步）。

**中階版**：

- 為某品牌做「重新設計」案例（建議挑你真的會用的台灣品牌：如某銀行 App、某外送、某學校選課系統）。
- 製作一份原創 icon set 並上架 Figma Community。
- 設計一份個人作品集網站。

**高階版**：

- 經營設計類 IG / Threads / X（原 Twitter），分享每週設計觀察。
- 開設線上設計課程或工作坊（可考慮 Hahow、PressPlay Academy 等台灣本地平台）。

### 想當行銷？

**簡單版**：經營一個 IG / Threads 帳號 3 個月，從 0 開始累積你能維持的追蹤數。

**中階版**：

- 為某品牌寫一份完整的 SEO 內容計畫。
- 做一個 Newsletter，每週分享一個產業洞察。
- 經營一個 TikTok / Threads 帳號。

**高階版**：

- 建立一個小型品牌（Side Brand），從產品設計、行銷到銷售完整經營（台灣大學生常見練習場：Pinkoi、蝦皮賣場、IG 小店）。

### 想當工程師？

**簡單版**：

- 做一個解你自己問題的 App / Chrome Extension / LINE Bot。
- 上傳到 GitHub，附完整 README。

**中階版**：

- 做一個能整合 LLM API（OpenAI、Anthropic Claude、Google Gemini 等，依當期可用模型為準）的小工具。
- 用 GitHub Actions 自動化一個流程。
- 開源一個小型 library。

**高階版**：

- 做一個有真實用戶的 Web 應用，並開放公開使用。
- 寫技術部落格（iThome 鐵人賽、個人 Hexo / Astro blog、Medium），深度分享技術選型與決策。

### 想當分析師？

**簡單版**：

- 用 Kaggle 或政府資料開放平台（data.gov.tw）公開資料做一份產業洞察。

**中階版**：

- 為一家上市公司做財務模型（公開資訊觀測站 mops.twse.com.tw 有完整財報）。
- 用 Tableau / Power BI / Looker Studio 製作互動儀表板。

**高階版**：

- 開設數據分析 Podcast 或 YouTube 頻道。
- 建立一個專注某產業的「資料儀表板」公開讓大眾使用。

## 五、給「沒時間」的學生

你不需要每天 4 小時做 side project。試試以下節奏：

### 每天 30 分鐘

- 早上 30 分鐘：寫一段 code、寫一篇短文、做一張設計稿。
- 累積一週 = 3.5 小時。
- 累積一個月 = 14 小時。

### 週末 2 小時

- 週六或週日 2 小時集中投入。
- 累積一個月 = 8–10 小時。

### 6 週為一個迭代

不要設定「我要做一個完美產品」，設定「我要在 6 週內做出 v1」。

### 每週固定發布一次進度

- 寫一篇 LinkedIn / Threads 貼文。
- 發 1 個 Medium / vocus 短文。
- 上傳 1 個進度截圖到 IG。

3 個月後，你會擁有比同學多 50 小時的「實戰肌肉」。

## 六、Side Project 的「最小可行版本」（MVP）思維

### 錯誤心態

「我要做一個完美的產品再上線。」

→ 結果：6 個月後還在做、沒上線。

### 正確心態

「我要先做一個粗糙但能用的版本，再迭代。」

→ 結果：1 個月後上線，根據用戶回饋持續改進。

> 這個觀念在《精實創業》（The Lean Startup，Eric Ries，行人出版社繁中譯本）與《Show Your Work!》（《點子就要秀出來》，Austin Kleon，遠流）等書中都有詳細討論。

### MVP 範例

**完美主義版**：

- 想做一個「大學生履歷生成器」。
- 計畫：完整的 UI、3 種模板、AI 推薦、排版優化、PDF 下載、多語...
- 結果：6 個月還沒完成。

**MVP 版**：

- v1：用 Google Doc 模板 + 一個 Notion 頁面。
- v2：開始用 React / Next.js 寫簡單版本。
- v3：加上 PDF 下載功能。
- v4：加入 AI 建議（接 LLM API）。

每個版本都能讓真實用戶使用。

## 七、把 Side Project 寫進履歷

> 以下三個範例為**示意虛構**，不是真實人物，數字只是展示「寫法的結構」。重點是學會用「行為 + 數字 + 影響」的格式來描述自己的 side project。

### 範例 1（工程師向，示意）：

> 「自製『大學生履歷模板生成器』Web App：
>
> - 使用 Next.js + LLM API 開發。
> - 累積數百名活躍用戶（具體數字請寫你實際達到的）。
> - 被 Dcard 相關看板或 Threads 設計創作者轉貼。
> - GitHub Stars: 寫實際數字。
> - 每週迭代，目前已發布 vN。」

### 範例 2（內容創作者向，示意）：

> 「個人 Newsletter《某產業觀察》：
>
> - 每週一發送，目前訂閱數寫實際數字。
> - 平均開信率寫實際數字（可附當期業界平均做參考；2025–2026 各類 Newsletter 開信率約落在 28–49% 區間，依平台與產業差異大，請以你使用的工具當期報告為準）。
> - 累積 N 篇文章，總字數 M 萬字。
> - 受邀某活動／某分享會（如有，且為真實）。
> - 最熱門文章被某媒體轉載（如有，且為真實）。」

### 範例 3（社群經營向，示意）：

> 「IG / Threads 帳號『某主題』：
>
> - 經營 N 個月，從 0 到 X 追蹤。
> - 每週 N 篇內容，平均互動率寫實際數字（IG 2025–2026 健康互動率約 1.5–3%，依帳號類型差異大）。
> - 與 N 個品牌合作付費業配（如有）。
> - 受邀至校園分享會（如有）。」

**履歷寫作原則**：千萬不要造假數字。真實的「100 個用戶」比虛構的「10,000 個用戶」可信百倍——面試官一問細節就會穿幫。

## 八、Side Project 的三個常見錯誤

### 錯誤 1：想做「驚人的東西」

反而會 1 年都沒做出來。**從小開始**。

許多 side project 失敗不是因為「不夠好」，是因為「沒做完」。

### 錯誤 2：自己一個人做

找夥伴會大幅提升完成率，且能模擬真實協作。

理想搭配：

- 你 + 1 位互補的夥伴（例：你做設計，他寫程式）。
- 每週固定 sync 一次。
- 互相 accountability。

**台灣大學生找夥伴的本地管道**：
- 學校創新創業中心（台大創創、政大商創、清大育成、成大育成、北科 EMI）。
- Facebook 社團、Dcard 創作者俱樂部、Threads 同主題標籤。
- 實習通、CakeResume Project 區、Yourator 社群活動。
- AppWorks School（程式 / 設計）、 ALPHA Camp 校友圈、Hahow 學習社群。

### 錯誤 3：做完不分享

分享是大半價值來源。沒分享 = 沒人知道你做了。

最低限度：

- LinkedIn / Threads 發一篇貼文。
- Medium / vocus 寫一篇文章。
- 加進個人作品集（GitHub Pages、Notion 公開頁、CakeResume 作品集區）。
- 真正想被看見：上 Product Hunt、Hacker News（Show HN）、iThome 鐵人賽——這些是台灣 indie 圈固定的曝光場。

## 九、Side Project 帶來的隱性禮物

### 1. 你會找到「我能持續做的事」

這對找方向超有幫助。如果你連 side project 都沒興趣繼續，這個方向就不適合你。

> 《原子習慣》（Atomic Habits，James Clear，方智出版社繁中譯本）整本書其實就是在講「身分認同 × 微小行動 × 複利」——side project 是這套理論最好的練習場。

### 2. 你會建立第一批粉絲與支持者

這是個人品牌的起點。少數真心粉絲 > 大量冷淡追蹤。

### 3. 你會增加「自我效能感」

完成一個專案後，你看自己的眼光不一樣了。「原來我可以做出這個」是大學期間最珍貴的體驗。

### 4. 你會學會「真實的時間管理」

平衡課業、實習、社交、休息與 side project，這個能力直接遷移到職場。

### 5. 你會擁有「對外發聲的肌肉」

寫文、發貼文、做分享——這些能力會幫你建立 5–10 年的個人品牌。

## 十、Side Project 的 6 個月成長軌跡（示意）

> 注意：以下時間軸是「中位數路徑」的示意，每個人實際進度差異極大，**不要把它當成 KPI**。沒達到不代表失敗。

### 第 1 個月：構想 + 開始

- 定義問題、目標用戶。
- 做 v1（最簡單的版本）。
- 找 5–10 個第一批用戶（朋友、同學、社團夥伴）。

### 第 2 個月：迭代 + 公開

- 根據回饋改進。
- 開始公開分享進度（IG / Threads / Medium）。
- 第一篇 Medium / vocus / 個人 blog 文章。

### 第 3 個月：成長 + 內容

- 開始累積真實使用數據。
- 每週 1 篇進度文。
- 開始看到第 N 個用戶（具體數字依題目與行銷力度差異大）。

### 第 4 個月：突破 + 收益（如果走到這步）

- 找到 product-market fit 的初步訊號。
- 可能開始有人主動聯繫你。
- 媒體曝光「不是常態，而是 bonus」。

### 第 5 個月：影響 + 擴展

- 用戶數成長。
- 寫一篇深度反思文章。
- 考慮收費或商業化（indie hacker 路線）。

### 第 6 個月：總結 + 下一步

- 寫一份完整的 retrospective（覆盤）。
- 決定要繼續、轉型、或結束（結束也是健康決定，不是失敗）。
- 把整個過程寫成案例放進作品集。

## 十一、Side Project 的「成功」如何定義？

### 不要只看用戶數

- 少量真心用戶 > 大量「殭屍追蹤」。
- 完成 v3、有持續迭代 > 一次性的爆紅。

### 看你的學習

- 你學會了什麼新技能？
- 你建立了什麼新習慣？
- 你的視野擴展了嗎？

### 看你的網絡

- 你認識了哪些新朋友？
- 你被哪些業界人士注意？
- 你的 LinkedIn / CakeResume 連結增加了多少？

### 看你的作品集

- 你能拿出來談的案例多了幾個？
- 你的「故事庫」豐富了嗎？

## 十二、給「不知道做什麼」的學生

### 用「30 分鐘清單」找靈感

列出：

- 你最近 1 個月「希望有人解決」的問題（5 個）。
- 你最常使用但覺得「應該更好」的工具（5 個）。
- 你最常聊到的話題或興趣（5 個）。

從這 15 個項目中選 1 個最簡單能做的。

### 範例：「我最近希望有人解決的問題」（台灣大學生情境）

- 找實習資訊太分散（橫跨 104、Yourator、CakeResume、實習通、各校就輔）。
- 系上同學的筆記沒地方共享。
- 找學長姐 Coffee Chat 沒有平台。
- 練習面試沒有對象。
- 想記錄學習歷程但沒工具。
- 校內食堂菜單 / 公車到站 / 借書到期沒有好用提醒。

每一個都可以是 side project 的起點。

## 十三、台灣本地 Side Project 啟發

> 以下是真實存在、且廣為人知的台灣 side project / indie 案例（截至 2026-05；最新動態請查各專案官網或社群）。

- **美玉姨**：由旅居國外的台灣工程師徐曦於 2018 年開發的 LINE 假訊息查證 Bot，從「家族群組長輩亂傳訊息」這個自己的痛點出發，後來累積數十萬好友。是「scratch your own itch」的代表案例。
- **iThome 鐵人賽**：台灣最大的工程師寫作社群挑戰，連續 30 天每天發文。許多大學生第一個被看見的 side project 都是從鐵人賽開始。
- **Hahow 好學校**、**PressPlay Academy**：台灣大學生與創作者開課的本地平台，是「把 side project 變成第一份收入」的常見路徑。
- **政府資料開放平台 data.gov.tw**：許多大學生的第一個資料 side project 都從這裡的公開資料集開始。
- **AppWorks Demo Day**、**Meet 創業小聚**：台灣本地創業／indie 圈固定曝光場合，部分有大學生組別。

## 十四、FAQ

**Q1：Side Project 一定要技術類嗎？**
不是。Newsletter、Podcast、IG / Threads 帳號、寫作、設計、資料分析、社群經營都算。**重點是「你主導從 0 到 1」**。

**Q2：可以團隊一起做嗎？**
可以，但要清楚自己的貢獻。寫履歷時要明確說「我負責 X、Y、Z」。

**Q3：Side Project 沒爆紅算失敗嗎？**
不算。**完成的失敗 > 沒開始的成功**。爆紅本來就是 outlier，不是 baseline。

**Q4：要不要把 side project 商業化？**
看你的目標：

- 想累積經驗：不必商業化。
- 想驗證商業思維：可以嘗試小額收費。
- 想走 indie hacker 路線：建議從第一天就思考 monetization。

**Q5：完整做完一個 side project 大約要多久？**
依複雜度：1 個月（簡單）到 1 年以上（複雜）。建議第一個 side project 控制在 1–3 個月內。

**Q6：用 AI 寫的 code 算 side project 嗎？**
算。2026 年用 Cursor / Claude Code / Copilot 已是業界常態，重點不是「你有沒有自己打每一行字」，而是「你能不能解釋為什麼這樣設計、知不知道哪裡會壞、能不能維護它」。面試官會問細節，假不來。

## 結語

Side project 是大學生最便宜、最不需要授權、最能改變履歷的活動之一。**你的限制不是時間，是你還沒開始**。

**核心原則**：

1. **從小開始**：不要追求完美，追求完成。
2. **公開分享**：不被看見就沒有複利。
3. **持續迭代**：每週 1 個小進步，累積驚人。
4. **建立習慣**：side project 是長期能力，不是一次性活動。

**行動建議**：今天就拿 30 分鐘列出 3 個你想做的小專案，挑一個最簡單的，這週末就開始第一個版本。完成後到實習通分享你的學習心得，讓更多學弟妹受惠。半年後，你會驚訝於自己的成長軌跡。
