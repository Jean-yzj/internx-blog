# Side Project 的力量：用業餘時間累積實戰經驗

「我沒有實習機會，怎麼累積經驗？」——這是大學生常見的焦慮。但其實，**Side Project（業餘專案）** 是過去十年最改變學生履歷的趨勢。許多頂尖工程師、設計師、PM 都是從一個小 side project 開始進入職場。本篇將帶你看懂 side project 的真正威力與實戰做法。

## 一、什麼是 Side Project？

任何「不是課業也不是工作」的個人專案，都算 side project。

### 例子
- 自己寫一個 App、網站、Bot。
- 經營一個 IG / Newsletter / Podcast。
- 為一個品牌做免費的設計提案。
- 解一份公開資料集做分析。
- 寫一篇深度的產業研究。
- 拍一個 YouTube 頻道。
- 開發一個 Chrome Extension。
- 做一個 Discord Bot。

### Side Project 與其他活動的差異

| 活動 | 主導者 | 時間 | 成果歸屬 |
|------|--------|------|----------|
| 課業 | 老師 | 學期 | 個人 |
| 實習 | 公司 | 數月 | 公司 |
| 社團 | 集體 | 學期 | 社團 |
| Side Project | **你自己** | 任意 | **你** |

關鍵在於「**你是發起者、決策者、執行者、收割者**」。

## 二、Side Project 為什麼比實習還有用？

### 1. 你是專案的擁有者
實習通常是執行別人定義好的任務，但 side project 是你自己定義問題、自己決策、自己交付。**這展現的是「主導力」**。

### 2. 它是你的長期資產
實習結束後，工作成果可能屬於公司，但 side project 永遠是你的。

具體價值：
- 5 年後你還能拿出來談。
- 可以放進個人網站、LinkedIn、Newsletter。
- 跨公司轉職時依然有效。

### 3. 它能跨越科系與背景
HR 不在乎你是哪個系，他在乎你做出了什麼。
- 中文系做出 1000 人使用的 App > 資工系沒做過任何專案。
- 哲學系經營 5,000 訂閱的 Newsletter > 商管系只有課業。

### 4. 它是「最低成本」的差異化
其他 100 位候選人都有實習，但只有你有「自己經營一年的 Newsletter」。

### 5. 它能讓你「真的會做事」
實習中你可能只做了某個小區塊。Side Project 你必須**從 0 到 1 完整執行**：
- 想 idea。
- 找用戶。
- 設計。
- 開發。
- 上線。
- 行銷。
- 維護。

這個全流程經驗，是實習給不了的。

### 6. 它能驗證你的職涯方向
想做 PM？做一個 App 看你能不能享受。
想做設計？做設計案例看你能不能持續。
想做寫作？經營 Newsletter 看你能不能輸出。

## 三、好的 Side Project 應有的 5 個特質

### 1. 真實解決一個問題
不要只是「我想做」，要有「為誰、為什麼」。

### 2. 可被驗證
有用戶、有數據、有回饋，能證明你的成果。

### 3. 能被簡單描述
1 句話講清楚（例如：「我做了一個幫大學生找實習的工具」）。

### 4. 可被分享
做出來後一定要公開，不要做完就丟到資料夾。**沒分享 = 沒做過**。

### 5. 能持續迭代
最好能維持 3–6 個月，看到成長曲線。

## 四、不同興趣的 side project 範例

### 想當 PM？

**簡單版（4 週）**：
- 為一款常用 App 寫「3 個改進建議」並發 Medium。

**中階版（2–3 個月）**：
- 用 Notion 模擬一份產品 Roadmap。
- 找 5 位用戶做訪談並輸出洞察報告。
- 為某產業寫一份完整的市場分析。

**高階版（6 個月+）**：
- 自己做一個小型工具，從用戶研究、設計、實作（找 no-code）、上線到推廣。

### 想當設計師？

**簡單版**：完成 Daily UI 100 天挑戰。

**中階版**：
- 為某品牌做「重新設計」案例。
- 製作一份原創 icon set 並上架 Figma Community。
- 設計一份個人作品集網站。

**高階版**：
- 經營設計類 IG / X（原 Twitter），分享每週設計觀察。
- 開設線上設計課程或工作坊。

### 想當行銷？

**簡單版**：經營一個 IG 帳號 3 個月，從 0 到 500 追蹤。

**中階版**：
- 為某品牌寫一份完整的 SEO 內容計畫。
- 做一個 Newsletter，每週分享一個產業洞察。
- 經營一個 TikTok 帳號，從 0 到 1000 追蹤。

**高階版**：
- 建立一個小型品牌（Side Brand），從產品設計、行銷到銷售完整經營。

### 想當工程師？

**簡單版**：
- 做一個解你自己問題的 App / Chrome Extension。
- 上傳到 GitHub，附完整 README。

**中階版**：
- 做一個能整合 OpenAI API 的小工具。
- 用 GitHub Action 自動化一個流程。
- 開源一個小型 library。

**高階版**：
- 做一個有真實用戶（500+）的 Web 應用。
- 寫技術部落格，深度分享技術選型與決策。

### 想當分析師？

**簡單版**：
- 用 Kaggle 公開資料做一份產業洞察。

**中階版**：
- 為一家上市公司做財務模型。
- 用 Tableau / Power BI 製作互動儀表板。

**高階版**：
- 開設數據分析 Podcast 或 YouTube 頻道。
- 建立一個專注某產業的「資料儀表板」公開讓大眾使用。

## 五、給「沒時間」的學生

你不需要每天 4 小時做 side project。試試以下節奏：

### 每天 30 分鐘
- 早上 30 分鐘：寫一段 code、寫一篇短文、做一張設計稿。
- 累積一週 = 3.5 小時。
- 累積一個月 = 14 小時。

### 週末 2 小時
- 週六或週日 2 小時集中投入。
- 累積一個月 = 8–10 小時。

### 6 週為一個迭代
不要設定「我要做一個完美產品」，設定「我要在 6 週內做出 v1」。

### 每週固定發布一次進度
- 寫一篇 LinkedIn 貼文。
- 發 1 個 Medium 短文。
- 上傳 1 個進度截圖到 IG。

3 個月後，你會擁有比同學多 50 小時的「實戰肌肉」。

## 六、Side Project 的「最小可行版本」（MVP）思維

### 錯誤心態
「我要做一個完美的產品再上線。」

→ 結果：6 個月後還在做、沒上線。

### 正確心態
「我要先做一個粗糙但能用的版本，再迭代。」

→ 結果：1 個月後上線，根據用戶回饋持續改進。

### MVP 範例

**完美主義版**：
- 想做一個「大學生履歷生成器」。
- 計畫：完整的 UI、3 種模板、AI 推薦、排版優化、PDF 下載、多語...
- 結果：6 個月還沒完成。

**MVP 版**：
- v1：用 Google Doc 模板 + 一個 Notion 頁面。
- v2：開始用 React 寫簡單版本。
- v3：加上 PDF 下載功能。
- v4：加入 AI 建議。

每個版本都能讓真實用戶使用。

## 七、把 Side Project 寫進履歷

### 範例 1：

> 「自製『大學生履歷模板生成器』Web App（2024）：
> - 使用 Next.js + OpenAI API 開發。
> - 3 個月內累積 1,200 名活躍用戶。
> - 被 Dcard 設計版精選為當週熱門。
> - GitHub Stars: 85。
> - 每週迭代，目前已發布 v8。」

### 範例 2：

> 「個人 Newsletter《Z 世代產業觀察》（2024–現在）：
> - 每週一發送，目前訂閱數 800+。
> - 平均開信率 42%（業界平均 21%）。
> - 累積 36 篇文章，總字數 18 萬字。
> - 受邀至某品牌實習招募分享會。
> - 最熱門文章被知名媒體轉載。」

### 範例 3：

> 「IG 帳號『_design_thinking_』（2024–現在）：
> - 經營 8 個月，從 0 到 5,200 追蹤。
> - 每週 3 篇內容，平均互動率 8.5%（業界平均 1–3%）。
> - 與 4 個品牌合作付費業配。
> - 受邀至 2 場校園分享會。」

## 八、Side Project 的三個常見錯誤

### 錯誤 1：想做「驚人的東西」
反而會 1 年都沒做出來。**從小開始**。

90% 的 side project 失敗不是因為「不夠好」，是因為「沒做完」。

### 錯誤 2：自己一個人做
找夥伴會大幅提升完成率，且能模擬真實協作。

理想搭配：
- 你 + 1 位互補的夥伴（例：你做設計，他寫程式）。
- 每週固定 sync 一次。
- 互相 accountability。

### 錯誤 3：做完不分享
分享是 50% 的價值來源。沒分享 = 沒人知道你做了。

最低限度：
- LinkedIn 發一篇貼文。
- Medium 寫一篇文章。
- 加進個人作品集。

## 九、Side Project 帶來的隱性禮物

### 1. 你會找到「我能持續做的事」
這對找方向超有幫助。如果你連 side project 都沒興趣繼續，這個方向就不適合你。

### 2. 你會建立第一批粉絲與支持者
這是個人品牌的起點。100 位真心粉絲 > 10,000 個冷淡追蹤。

### 3. 你會增加「自我效能感」
完成一個專案後，你看自己的眼光不一樣了。「原來我可以做出這個」是大學期間最珍貴的體驗。

### 4. 你會學會「真實的時間管理」
平衡課業、實習、社交、休息與 side project，這個能力直接遷移到職場。

### 5. 你會擁有「對外發聲的肌肉」
寫文、發貼文、做分享——這些能力會幫你建立 5–10 年的個人品牌。

## 十、Side Project 的 6 個月成長軌跡

### 第 1 個月：構想 + 開始
- 定義問題、目標用戶。
- 做 v1（最簡單的版本）。
- 找 5–10 個第一批用戶。

### 第 2 個月：迭代 + 公開
- 根據回饋改進。
- 開始公開分享進度。
- 第一篇 Medium 文章。

### 第 3 個月：成長 + 內容
- 開始累積真實使用數據。
- 每週 1 篇進度文。
- 開始看到第 100 個用戶。

### 第 4 個月：突破 + 收益
- 找到 product-market fit 的訊號。
- 開始收到媒體、業界注意。
- 可能有人主動聯繫你。

### 第 5 個月：影響 + 擴展
- 用戶數突破 500–1000。
- 寫一篇深度反思文章。
- 考慮收費或商業化。

### 第 6 個月：總結 + 下一步
- 寫一份完整的 retrospective。
- 決定要繼續、轉型、或結束。
- 把整個過程寫成案例放進作品集。

## 十一、Side Project 的「成功」如何定義？

### 不要只看用戶數
- 100 個真心用戶 > 10,000 個假數字。
- 完成 v3、有持續迭代 > 一次性的爆紅。

### 看你的學習
- 你學會了什麼新技能？
- 你建立了什麼新習慣？
- 你的視野擴展了嗎？

### 看你的網絡
- 你認識了哪些新朋友？
- 你被哪些業界人士注意？
- 你的 LinkedIn 連結增加了多少？

### 看你的作品集
- 你能拿出來談的案例多了幾個？
- 你的「故事庫」豐富了嗎？

## 十二、給「不知道做什麼」的學生

### 用「30 分鐘清單」找靈感
列出：
- 你最近 1 個月「希望有人解決」的問題（5 個）。
- 你最常使用但覺得「應該更好」的工具（5 個）。
- 你最常聊到的話題或興趣（5 個）。

從這 15 個項目中選 1 個最簡單能做的。

### 範例：「我最近希望有人解決的問題」
- 找實習資訊太分散。
- 系上同學的筆記沒地方共享。
- 找學長姐 Coffee Chat 沒有平台。
- 練習面試沒有對象。
- 想記錄學習歷程但沒工具。

每一個都可以是 side project 的起點。

## 十三、FAQ

**Q1：Side Project 一定要技術類嗎？**
不是。Newsletter、Podcast、IG 帳號、寫作都算。**重點是「你主導從 0 到 1」**。

**Q2：可以團隊一起做嗎？**
可以，但要清楚自己的貢獻。寫履歷時要明確說「我負責 X、Y、Z」。

**Q3：Side Project 沒爆紅算失敗嗎？**
不算。**完成的失敗 > 沒開始的成功**。

**Q4：要不要把 side project 商業化？**
看你的目標：
- 想累積經驗：不必商業化。
- 想驗證商業思維：可以嘗試小額收費。

**Q5：完整做完一個 side project 大約要多久？**
依複雜度：1 個月（簡單）到 1 年（複雜）。建議第一個 side project 控制在 1–3 個月內。

## 結語

Side project 是大學生最便宜、最不需要授權、最能改變履歷的活動。**你的限制不是時間，是你還沒開始**。

**核心原則**：
1. **從小開始**：不要追求完美，追求完成。
2. **公開分享**：50% 的價值在「被看見」。
3. **持續迭代**：每週 1 個小進步，累積驚人。
4. **建立習慣**：side project 是長期能力，不是一次性活動。

**行動建議**：今天就拿 30 分鐘列出 3 個你想做的小專案，挑一個最簡單的，這週末就開始第一個版本。完成後到實習通分享你的學習心得，讓更多學弟妹受惠。半年後，你會驚訝於自己的成長軌跡。
