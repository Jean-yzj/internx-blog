# Slack / Teams 職場禮儀：傳訊息也是專業表現

「我傳訊息給主管，他半小時都不回，我要再傳一次嗎？」「同事都在用 emoji 反應我的訊息，我也要這樣回嗎？」「為什麼我傳了長訊息卻沒人讀？」即時通訊工具像 Slack、Teams、Discord，已經是台灣多數企業的內部主要溝通管道，比 email 更頻繁、更非正式，但也更容易踩雷。傳訊息的方式，其實隱藏著公司文化、團隊權力結構、甚至個人品牌訊號。本篇文章帶你拆解頻道使用、@提及禮儀、表情符號、async 工作文化，以及「什麼時候該用 Slack、什麼時候要轉 email、什麼時候要打電話」的實戰判斷。

## 一、為什麼即時通訊比 email 更難拿捏

Email 有清楚的結構與正式格式，Slack / Teams 卻是「介於聊天與工作之間」的灰色地帶：

- 太正式 → 看起來不合群
- 太隨便 → 看起來不專業
- 太頻繁 → 打擾別人
- 太安靜 → 看起來不在線

實習生最常踩的雷是「把 Slack 當 LINE 用」或「把 Slack 當 email 用」。其實它有自己的潛規則。

## 二、頻道使用的 5 個原則

### 1. 先讀懂頻道的「定位」
進公司後第一週，先把所有公開頻道掃過一遍，搞清楚：

| 頻道類型 | 用途 | 發言頻率 |
|---------|------|---------|
| `#announcements` | 公司公告 | 不要發，只讀 |
| `#general` | 全公司閒聊 | 偶爾參與，不要發工作問題 |
| `#team-OO` | 部門工作 | 主要工作場域 |
| `#proj-OO` | 專案討論 | 集中在這裡 |
| `#random` / `#fun` | 純閒聊 | 自由參與 |
| `#help-OO` | 求助頻道 | 問問題前先搜尋 |

### 2. 公開 vs 私訊的判斷
**用公開頻道**：
- 問題對其他人也有參考價值
- 需要團隊知道進度
- 想被「順便看到」的小成就

**用私訊**：
- 個人問題（請假、生理狀況、家庭）
- 對某人的回饋（尤其是負面）
- 還在發想階段、不想公開的想法

### 3. 不要在 #general 問工作問題
全公司頻道是用來公告，不是用來私聊或求助。把問題發到對的小頻道，得到答案的速度會快十倍。

### 4. Thread（討論串）是你的好朋友
回應某條訊息時，**永遠用 thread 回覆**，不要在主頻道單獨發一條。Thread 可以避免頻道被洗版，也讓相關訊息聚在一起。

### 5. 開新頻道前先確認沒有重複
看到一個議題沒人在討論，先搜尋有沒有現成頻道。亂開新頻道是新人最容易得罪人的事。

## 三、@提及（Mention）的禮儀

@提及在 Slack / Teams 等於「拍對方一下」，是最容易激怒同事的功能。

### 1. @提及的層級
- `@channel`：通知頻道內所有人，**幾乎不要用**，除非真正全員相關
- `@here`：通知當下在線的人，比 @channel 溫和，但仍要謹慎
- `@用戶名`：點對點通知，最常用

### 2. 什麼時候該 @某人
- 你需要他在 24 小時內回應
- 訊息內容直接跟他相關
- 你問了一般性的問題沒人回，需要點名

### 3. 什麼時候不要 @
- 純資訊分享，不需要對方回應
- 你只是引述某人說過的話
- 對方剛回完訊息，又 @ 他催第二次（不耐煩）

### 4. @了之後對方不回怎麼辦
- 1 小時內：耐心等，他可能在會議
- 4 小時內：不要再 @ 同一條訊息，可以發新訊息「補充說明」
- 一整天：考慮是否該改用 email、找他本人、或繞道（找他的同事）

### 5. 避免「殭屍 @」
有些實習生喜歡 @ 主管 + @ 同事一大串，這在主管眼裡是「你不會獨立判斷誰該負責」。@ 應該精準到只有「真正需要回應的那個人」。

## 四、Emoji 與表情符號的潛規則

很多實習生不知道 emoji 在職場有訊號意義。

### 1. Emoji 反應（reaction）的用法

| Emoji | 通常意思 |
|-------|---------|
| ✅ / 👍 | 收到、同意、確認 |
| 👀 | 我看到了，正在處理 |
| 🙏 | 感謝、拜託 |
| 🎉 / 🎊 | 慶祝、恭喜 |
| ❤️ / 💯 | 讚、強烈同意 |
| 🤔 | 我有疑問（要小心，可能被解讀為質疑） |
| 😅 | 尷尬，最好少用 |

### 2. 用 emoji 取代「收到！」
如果同事傳給你一個任務，與其回「收到，我會處理」，可以直接 reaction 一個 ✅ 或 👀，省下對方的閱讀時間。這是 Slack 文化的精髓。

### 3. Emoji 數量的潛規則
- 對主管：每則訊息 0–1 個 emoji
- 對平輩：可 1–2 個
- 對下屬 / 同事熟人：可隨意

### 4. 文化差異要敏感
- 拇指 👍 在某些國家有負面意涵，跟外國同事溝通要謹慎
- 表情符號跟字面意思不同（例如 💀 在新世代代表「笑死」）
- 跟資深同事用較保守的 emoji

## 五、長訊息 vs 短訊息：選對節奏

### 1. 一行一條 vs 整段一條
**Bad（洗版）：**
```
小美 1:23pm
你好
小美 1:23pm
我有問題想問
小美 1:23pm
是關於 X 專案
小美 1:23pm
你方便嗎？
```

**Good（一條完整）：**
```
小美 1:23pm
你好，想問一下 X 專案的 Y 部分，
你下午有 5 分鐘可以聊嗎？
```

切勿把 Slack 當 LINE 用，每句話一條訊息會讓對方手機震個不停。

### 2. 長訊息要先給結論
超過 100 字的訊息，第一行就要寫結論：

```
【請示】X 專案下週是否上線？

背景：
昨天會議結論是 6/15 上線，但今天看到競品延後到 6/20。
我們是否要等競品先發再跟進？

兩個選項：
A. 維持 6/15 上線（搶先）
B. 延後到 6/22（避免被比較）

你傾向哪一個？
```

### 3. 表格、條列、程式碼區塊
Slack / Teams 都支援 Markdown，善用：
- 條列（`-` 或 `*`）讓重點清楚
- 程式碼區塊（` ``` `）讓程式碼不變形
- 粗體（`*粗體*`）強調重點

不要用 emoji 排版，看起來像個人 IG。

## 六、Async 工作文化：實習生最該理解的概念

「Async」（非同步）是現代職場的核心觀念，意思是：**訊息發出後不期待對方立刻回應**。

### 1. Async 思維的核心
- 我傳訊息 ≠ 對方馬上要看
- 對方沒馬上回 ≠ 他不重視
- 你應該預期對方在 4–24 小時內回覆

### 2. 寫 async 訊息的技巧
- 把所有對方需要的資訊一次給齊（不要擠牙膏）
- 提供選擇題而不是申論題
- 給對方一個合理的回覆截止時間

**Bad（async 不友善）：**
```
你有空嗎？
```

**Good（async 友善）：**
```
你方便看一下這份簡報嗎？
重點 P3、P7 想請你確認方向。
如果你今天下午之前能回，我可以準時印製；
如果今天回不了，明天早上 10 點前也 OK。
```

### 3. 如何讓自己變成 async-friendly 的人
- 把通知時間設定在工作時間內
- 下班後設「勿擾」狀態
- 回應前先讀整段，不要片段回

## 七、什麼時候該用 Slack？什麼時候轉 Email？什麼時候打電話？

| 場景 | 工具 | 為什麼 |
|------|------|-------|
| 快速問題、確認小事 | Slack | 不需要正式紀錄 |
| 需要長期 reference 的決策 | Email | 容易找回、可歸檔 |
| 對外（客戶、廠商）溝通 | Email | 正式、可 cc 主管 |
| 跨部門、跨層級的正式請示 | Email | 留下軌跡 |
| 緊急問題（30 分鐘內要解決） | 打電話 / 直接走過去 | Slack 不保證讀取 |
| 情緒議題、敏感反饋 | 面對面 / 視訊 | 文字會被誤讀 |
| 需要快速協作的腦力激盪 | 視訊 + 共筆 | 有限度同步 |

### 重要原則：升級而不降級
如果一個議題在 Slack 已經來回 5 次以上還沒解決，**立刻升級到視訊或當面討論**，不要繼續打字。Slack 適合決策後的同步，不適合決策中的辯論。

## 八、緊急狀況的工具選擇

### 1. 主管沒回 Slack 但事情很急
- 第一步：傳第二條訊息（在 thread 裡）「這件事 30 分鐘內要決定，是否方便電話？」
- 第二步：傳簡訊或打電話
- 第三步：問同事「主管現在在哪裡？」

### 2. 客戶突然有狀況
- 立刻電話或當面找主管，不要單純 Slack 報告
- 有任何承諾都先回到主管確認後再回客戶

### 3. 系統故障、網站當機
- 直接走過工程主管的位子
- 同時在 #incident 頻道公告
- 不要只是 Slack，會被淹沒

## 九、回應時間期待表

不同層級、不同情境，回應時間的期待不同。

| 訊息來源 | 工作時間內 | 工作時間外 |
|---------|-----------|-----------|
| 直屬主管 @ 你 | 15–30 分鐘內回 | 不必立刻回，隔天上班 1 小時內 |
| 直屬主管的訊息（非 @） | 1 小時內看到，2 小時內回 | 隔天再回 |
| 客戶 | 30 分鐘內 ack（即使還沒答案） | 看公司文化，多數允許隔天 |
| 平輩同事 | 半天內 | 隔天 |
| 廣發公告 | 不必回 | 不必回 |
| 跨部門協作 | 4 小時內 | 隔天 |

## 十、FAQ：實習生最常問的 5 個 Slack 困惑

**Q1：主管 9 點傳訊息給我（我已下班），我要立刻回嗎？**
看訊息內容。如果是「明天記得 X」這種非緊急訊息，可以隔天上班再回。如果是「今晚 11 點客戶會問」，立刻回。一個原則：先回「我看到了」，再判斷怎麼處理。

**Q2：Slack 上同事用 emoji 開玩笑，我可以跟著玩嗎？**
觀察一週再加入。先看誰跟誰玩、玩什麼笑話、踩雷的人後來怎麼樣。融入文化要慢慢來，不要第一週就跟資深同事 hahaha。

**Q3：我發現自己被踢出某個頻道，怎麼辦？**
不要立刻問。先確認是不是頻道整體調整（例如改私密）。若只有你被踢，私訊主管「我注意到我被移出 X 頻道，是組織調整還是我有需要調整的地方？」用中性語氣詢問。

**Q4：DM 主管比較好還是用公開頻道？**
- 純個人問題、敏感議題：DM
- 工作進度、團隊相關：公開頻道（讓其他人也能參考）
- 不確定時：先 DM 問「這個我直接在頻道問可以嗎？」

**Q5：可以在 Slack 拒絕主管嗎？**
可以，但要給選項。「這週 X 任務已經滿了，你給我的這個新任務 Y，要我延到下週還是先暫停 X？」用選擇題的方式呈現衝突，不要直接說「我做不了」。

## 十一、結語：傳訊息是 21 世紀的職場素養

對你父母輩的職場人來說，Email 是專業力指標；對你這代來說，Slack / Teams 才是。傳訊息的清晰度、節奏感、政治敏感度，會決定主管想不想把更重要的對話框交給你。每一條訊息都是你的個人品牌建立。

**請記得三個關鍵原則**：
1. **每條訊息都先想「對方收到後要做什麼」**：不確定就不要傳。
2. **Async 思維是現代職場的禮貌**：給對方選擇何時回應的權利。
3. **越資深的人越不耐煩冗長訊息**：把結論放在第一行。

**行動建議**：到實習通搜尋你公司的內部溝通文化心得，注意前輩有沒有提到「Slack 文化」「會議節奏」「回訊速度」。不同公司差異極大——有些 24 小時都有人回、有些九點後絕不打擾。提前理解你即將進入的節奏，可以讓你的訊息禮儀第一週就到位，而不是慢慢踩雷學會。
