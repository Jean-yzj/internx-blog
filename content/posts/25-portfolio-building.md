# 自學專案怎麼做？打造個人作品集吸引面試官

「履歷只能寫經驗，但作品集能讓面試官看到你的能力。」這是越來越多 HR 在篩選大學生時的真實邏輯。一份好的作品集，比 GPA 更能展示你「動手做」的能力。本篇將完整教你如何從零開始打造一份能吸引面試官的個人作品集。

> **時點提醒**：以下列出的平台 / 工具請以 2026 年當期官方頁面為準。**平台關站 / 改名為常態**——本文也是因為發現某個曾被廣泛推薦給工程師的個人主頁平台已於 2025-05 關站才重新審訂。

## 一、為什麼學生需要作品集？

### 1. 補足缺乏正職經驗的不足

你還沒有 5 年工作經驗，但你可以有 5 個高品質作品。

### 2. 展現你的思考過程

作品集不只給結果，更給「你怎麼解決問題」的過程。HR 看到的不是「你會什麼」，而是「**你怎麼想、怎麼做**」。

### 3. 有別於其他候選人

履歷大同小異，但作品集獨一無二。一份精緻的作品集能讓你在 100 位候選人中脫穎而出。

### 4. 提升面試對話的深度

有作品集的面試 = 從「你做過什麼」變成「我看到你做的這個專案，能聊聊細節嗎？」。這是 100 倍的對話深度。

### 5. 累積長期資產

作品集可以一直更新、一直成長。5 年後它會成為你的個人品牌核心。

## 二、不同職位的作品集形式

### 1. 設計類（UI / UX / 品牌設計）

**國際平台**：Behance、Dribbble、個人網站。

**TW 學生實際也會用**：個人 IG（@username 形式作為 visual identity portfolio feed）、CakeResume 設計師檔案、Medium 中文設計過程文章、學校畢業展網站（實踐、台科大、北科大、雲科大畢展皆有公開站台）。

**內容**：4–6 個案例，每案例含問題定義、設計過程、最終成品。

**範例結構**：

- Cover image（一張代表作）。
- Project Overview（背景）。
- Problem Statement（問題）。
- Research（研究方法、洞察）。
- Design Process（流程、迭代）。
- Final Solution（成品）。
- Reflection（學到什麼）。

### 2. 工程類

**TW 學生實際組合**：GitHub（含 Pinned Repos 與 **Profile README** 自介）+ 個人網站（Next.js / Astro，部署在 Vercel、Netlify 或 **Zeabur**（台灣團隊出品的部署平台，前後端 + DB 一鍵部署，TW 學生圈使用率上升））+ **CakeResume 工程師檔案**（TW 新鮮人最常被 HR 翻到的平台之一）+ LinkedIn。

**內容**：3–5 個 Repo，每個 Repo 有清楚的 README、Demo 連結。

**範例結構**：

- README 包含：專案介紹、技術 stack、安裝、使用方式、Demo GIF。
- Code 結構清楚、有註解。
- Live Demo 連結。
- Architecture 圖（複雜專案）。

### 3. PM / 商業分析類

**平台**：Medium、個人 Notion、個人部落格（Hashnode、自架 Next.js）。

**TW 在地補充**：產品三眼怪實驗室、Mr.PM 等 TW PM 圈個人 Notion 公開頁是常見的範本可參考。

**內容**：產品分析、競品研究、PRD 範例、案例分析。

**範例結構**：

- Background（產品背景）。
- Problem（你發現的問題）。
- Research（用戶研究、競品分析）。
- Solution（你的提案）。
- Metrics（如何衡量成功）。

### 4. 行銷 / 內容類

**平台**：個人 IG、Newsletter（Substack、Buttondown）、Medium、官方部落格。

**內容**：實際操作數據、SEO 文章、廣告案例。

**範例結構**：

- Campaign Overview（活動概述）。
- Strategy（策略思考）。
- Execution（執行）。
- Results（量化成效）。
- Learnings（學到什麼）。

### 5. 數據分析類

**平台**：Tableau Public、GitHub、Kaggle。

**2026 年常見補充**：Google Colab 公開 Notebook、Hugging Face Spaces（ML / AI 模型互動 demo）、Streamlit Cloud（資料應用 demo）。

**內容**：3–5 個數據視覺化專案、商業洞察分析。

**範例結構**：

- Dataset（資料來源）。
- Question（要回答什麼問題）。
- Methodology（分析方法）。
- Insights（洞察）。
- Visualization（圖表）。

## 三、作品集結構黃金模板

每個案例都應有六段結構：

### 1. 背景（Context）

這是什麼專案？為什麼做？為誰做？

- 50–100 字。
- 設定情境、引導讀者。

### 2. 問題（Problem）

你想解決什麼？

- 用具體數據或現象描述。
- 為什麼這個問題重要？

### 3. 過程（Process）

你怎麼研究、怎麼決策？

- 訪談記錄。
- 競品分析。
- 假設與驗證。

### 4. 解方（Solution）

你做出了什麼？

- 視覺化呈現。
- 設計稿、原型、文件。

### 5. 成果（Result）

有什麼可量化的影響？

- 用戶數、互動率、留存率。
- 客戶 / 用戶回饋。

### 6. 反思（Reflection）

你學到什麼？下次會怎麼改？

- 誠實的自我評估。
- 接下來想嘗試的方向。

## 四、學生如何從零開始？

### 方法 1：把學校專題變成作品

不要讓辛苦做的期末報告埋沒在 Google Drive。整理成可分享的網頁版。

具體做法：

- 用 Notion / Webflow / Framer 重新編排。
- 加入過程素材（訪談紀錄、草稿、流程圖）。
- 補上「如果重做我會怎麼改」。

### 方法 2：模擬企業案例（用 TW 你日常使用的產品）

不需要等別人給你題目，自己創造。**選 TW 你真的常用的產品，代入感與面試共鳴會強很多**：

- 為 **Dcard / Threads / KKday / foodpanda / 蝦皮 / momo / LINE** 等你日常使用的 TW App 寫「重新設計提案」。
- 為 **誠品 / 王品集團 / 全聯 / 星巴克 TW** 等本土品牌寫「3 個月行銷計畫」。
- 為 **半導體 / 共享機車（GoShare / WeMo）/ 文創 / 夜市經濟** 等 TW 特色產業做「競品分析報告」。
- 為一個流行話題寫「使用者研究報告」。

### 方法 3：自學 + 實作

- 修完線上課程後，立刻做 1 個專案應用所學。
- 例：學完 SQL → 用公開資料集寫一份分析報告。
- 學完 Figma → 重新設計一款 App。

### 方法 4：紀錄真實兼職 / 社團經驗

即使是社團經驗，也能整理成案例（含目標、執行、成果）。

### 方法 5：發起一個小型社群活動

- 自辦讀書會、寫作群、產品分析會。
- 紀錄整個過程。
- 把它變成一個 case study。

## 五、提升作品集的五個細節

### 細節 1：第一頁就要 hook

第一個畫面要讓人在 5 秒內看出「你會什麼」。

具體做法：

- 用一張代表作圖片。
- 加上一句強烈的 tagline（「我把混亂變清晰」、「我用資料說故事」）。
- 列出 3 個你最強的能力。

### 細節 2：視覺一致

字型、顏色、版面風格統一，整體看起來才專業。

工具：

- Notion、Webflow、Framer 都有現成模板。
- Canva 製作封面圖。
- Figma 設計個人視覺識別。

### 細節 3：案例排序

最強的作品放最前面，弱的不要放。寧缺勿濫。

排序原則：

- 第一個：最 impressive、最有量化成果。
- 第二個：最近期、最有 relevance。
- 第三個：最能展現多元性的。
- 後面：補強其他面向。

### 細節 4：加入過程素材

草稿、流程圖、訪談筆記，這些能讓你顯得真誠。

- 設計稿的 v1、v2、v3。
- 用戶訪談的引言（去識別化）。
- 決策時的取捨討論。

### 細節 5：每個案例都有 1 個量化成果

即使是個人專案，也找一個指標衡量影響：

- 流量（獨立訪客）。
- 互動（評論、分享）。
- 留存（使用者回流）。
- 滿意度（用戶回饋）。

## 六、作品集的常見地雷

### 地雷 1：只放成品，沒有過程

HR 看不到你的思考。

### 地雷 2：過度華麗的動畫

載入慢、看不清重點。**內容 > 視覺特效**。

### 地雷 3：長篇文字沒分段

讀者會直接跳走。每 3–5 行一段、用標題分區。

### 地雷 4：模糊的「我參與了一個專案」

要清楚寫出你的具體角色。

- ❌「我們團隊做了 X。」
- ✅「我負責 X 的 A、B、C 部分，包含...」

### 地雷 5：缺少聯絡方式

找不到你怎麼錄取你？

- Email（必備）。
- LinkedIn 連結。
- 個人 IG / X（原 Twitter）/ Threads（如果經營得好）。

### 地雷 6：使用過時設計

2018 年的設計風格放在 2026 年看會很奇怪。每年至少更新一次風格。

### 地雷 7：版權問題

不要直接放公司內部作品（除非已經公開）。模擬案例要清楚標明「個人練習作品」。

## 七、作品集要放在哪裡？

最理想的策略是「**主站 + 平台分發**」：

### 主站

- 個人網站（Notion、Webflow、Framer 都好）。
- 一個簡單的 URL（你的名字 .com 或 .me）。
- 完整的個人介紹、作品、聯絡方式。

### 分發（TW 求職者建議 CakeResume + LinkedIn 雙軌並行）

**TW 必備**：

- **CakeResume**（TW 新鮮人 / 學生最常被 HR 搜尋到的履歷平台，工程、設計、PM 都用）。
- **Yourator**（新創、數位、設計、行銷職缺）。
- **104 / 1111 履歷**（傳產與中大型企業）。

**國際 / 業界補充**：

- LinkedIn（必備，外商 / 國際職缺）。
- Behance（設計類必備）。
- Dribbble（設計類）。
- GitHub（工程類必備）。
- Medium（PM、行銷、寫作）。
- 個人 IG / Threads（視覺、創意）。

### 在履歷上呈現

- 加一個 QR Code 或短網址。
- 把連結放在頂部（Email 旁邊）。
- HR 一秒打開。

## 八、給「我沒東西可放」的學生

從以下 5 個地方開始（**TW 大學生友善版本**）：

### 1. 一份你做過的學校報告

重新整理成有設計感的版本。

### 2. 一個你常用 App 的改進提案

找你最熟的 TW App（Dcard、Threads、KKday、foodpanda），列 5 個改進點，附上設計稿或文字提案。

### 3. 一份你目標公司的競品分析

深入研究 3–5 家競品，做完整分析。

### 4. 一個你 30 天內可完成的小工具或內容

- 在 **Dcard / Threads / Medium** 寫 4 篇深度心得（TW 大學生圈擴散高）。
- 經營一個 IG / Threads 帳號分享 TW 校園 / 設計 / 求職題材。
- 用 Next.js + Vercel / Zeabur 做一個解決 TW 大學生痛點的小工具（選課輔助、實習職缺聚合、課程評價等）。

### 5. 你管理過的 IG / 社團活動數據

即使是社團 IG，也能用截圖呈現成長軌跡。

**作品集不是「等夠厲害再做」，而是「邊做邊變厲害」**。

## 九、不同階段的作品集策略

### 大一 / 大二（建立期）

- 1–2 個案例就夠。
- 重點是「**有作品集這件事**」。
- 用簡單的 Notion 頁面開始。

### 大三（成長期）

- 累積到 5–6 個案例。
- 開始有自己的設計風格。
- 加入實習成果（去識別化後）。

### 大四（精煉期）

- 精選 5–8 個最強的案例。
- 升級到專業網站（Webflow / Framer）。
- 確保所有連結、動畫都運作正常。

## 十、作品集的「故事性」

最厲害的作品集不是「優秀的個別案例」，是「能串起一個故事的整體」。

### 範例：成長軌跡

> 「我的作品集記錄了從『學設計』到『懂用戶』的轉變：
> 第 1–2 個案例：純粹的視覺練習。
> 第 3–4 個案例：開始加入用戶研究。
> 第 5–6 個案例：完整的 UX 流程，從研究到驗證。
> 這個轉變反映了我這 3 年的學習與成長。」

### 範例：主題一致

> 「我的作品集都圍繞『教育科技』：
> 案例 1：為某線上學習平台做改進提案。
> 案例 2：自製一個學習計畫工具。
> 案例 3：訪談 30 位學生的學習痛點研究。
> 案例 4：設計一個學生互助平台原型。
> 這些作品展現我對這個領域的長期投入。」

## 十一、進階：作品集 + 個人品牌

當你有完整作品集後，下一步是建立「個人品牌」。

**TW 大學生實際的個人品牌通路（按擴散速度排序）**：

| 通路 | 特色 |
|---|---|
| **Threads / Dcard / IG** | TW 同儕圈擴散最快、學生社群密集 |
| **Medium / 個人部落格** | 深度內容、SEO 長尾、業界前輩會搜到 |
| **LinkedIn** | 外商 / 國際職缺、業界前輩主動聯絡 |
| **Newsletter（Substack / Buttondown）** | 受眾較小但忠誠度高，週期較長 |

**建議從 1–2 個通路深耕，不要五個都做**。

### 1. 在 LinkedIn 寫案例

把你的作品集案例改寫成 LinkedIn 貼文。**持續產出 3–6 個月後通常會有業界前輩或 HR 主動聯絡，但速度視內容品質、領域熱度、既有人脈而定，差異可達數倍**——重點是「持續」而非「多快」。

### 2. 在 Medium 發深度文章

每月 1 篇產業觀察。**累積到一定量後可能被產業前輩注意**，但同樣視內容品質與主題稀缺度而定，不要把「多久會有人看」當成單一指標。

### 3. 演講與分享

參加學校或產業的分享會，分享你的學習過程。

### 4. 開設 Newsletter

建立讀者群。TW 中文 Newsletter 受眾較小，**目標應是『穩定的 100 位真正會讀的讀者』，而非『1000 位掛名追蹤者』**。

## 十二、推薦工具與資源

### 作品集網站工具

- **Notion**：最快入門。
- **Webflow**：高度客製化。
- **Framer**：適合互動性強的作品集。**在校學生可申請 Framer for Education，免費取得 Basic Plan 全功能 + custom domain（需學校信箱驗證）**。
- **Cargo**：適合創意人。

### 設計師專屬

- **Behance**：Adobe 旗下，設計師大本營。
- **Dribbble**：設計社群、找工作。
- **Awwwards**：頂尖設計案例參考。
- **靈感蒐集**：Pinterest、IG 收藏夾、Mymind（TW 設計學生實際更常用，Are.na 偏歐美創意圈）。

### 工程師專屬

- **GitHub**：程式碼必備。**善用 Profile README**（在 `username/username` repo 放 README.md，會顯示在你的 GitHub 個人頁面，相當於免費的工程師個人主頁）。
- **Vercel / Netlify**：免費 hosting（個人作品集免費額度足夠）。
- **Zeabur**：**台灣團隊出品**的部署平台，支援前後端 + DB 一鍵部署，免費額度足夠個人作品集，UI 與文件對中文使用者友善。
- **CakeResume 工程師檔案**：TW 工程實習 / 新鮮人職缺最常被 HR 搜尋到的平台。

> **平台關站警示**：曾被廣泛推薦給工程師的 `Read.cv` 已於 **2025-05-16 正式關站**（團隊加入 Perplexity）。**這是 AI 寫作最常見的 stale 錯誤**——LLM 仍以舊資料推薦已不存在的平台。**任何作品集教學文，請以當期能打得開的官方頁面為準**。

### 創意人士

- **Notion + Super.so**：簡單個人網站。

## 十三、2026 補充：AI 工具如何加入作品集

**2026 年的作品集流程已經不能假裝 AI 不存在**，但 AI 是放大器、不是替代品。

### 大學生實際可用的 AI 工具

- **UI Prototype**：v0.dev、Lovable、Bolt.new、Vercel v0（用一句話生成可運行的 React UI，加速從想法到 demo 的速度）。
- **設計輔助**：Framer AI、Figma Make（從文字描述生成 UI 草稿）。
- **文案 / Storytelling**：GPT-5.5、Claude Opus 4.7 可用於草擬案例 storytelling、英文版翻譯、README 改寫。

### 但作品集的核心仍要自己產出

問題定義、設計決策、為什麼選 A 不選 B、用戶訪談的 insight——**這些是面試官會追問的部分**。照搬 AI 輸出反而會在面試時露餡：

> 面試官：「這個按鈕為什麼放右下角不放右上角？」
>
> 你：「呃...」（因為你沒想過，是 AI 直接生的）

**建議用法**：AI 加速 prototype 與排版、加速文字草稿，但每個關鍵決策都要能自己解釋。

## 十四、FAQ

**Q1：作品集要寫多長？**
精選 3–6 個最強案例，每案例詳略視職位調整（設計類重圖少字、工程類重 code / 架構、PM 重思考過程）。整體應讓 HR 能在 5–10 分鐘看完核心故事，深入閱讀則 15–20 分鐘。

**Q2：要英文還是中文？**
看目標：

- 投本土公司：中文為主。
- 投外商：英文必備。
- 雙語版本最理想。TW 部分本土科技公司（Appier、iKala、KKday 等）即使是中文職缺，HR / Hiring Manager 也可能會看英文版以評估國際溝通能力。

**Q3：可以放在校期間的學分作業嗎？**
可以。重新整理、加入過程、補上反思即可。

**Q4：要付費請人做嗎？**
不必。免費工具（Notion、Framer Education 學生方案、Vercel / Netlify / Zeabur 免費 hosting）+ 一個下午就能做出基本版本。

**Q5：作品集多久更新一次？**
每 3 個月小更新（新增案例），每年大更新（重新設計風格）。**另外建議每半年檢查一次平台是否仍在營運**——Read.cv 在 2025-05 關站就是血淋淋的例子。

## 結語

履歷是你過去的總結，作品集是你能力的證明。**你不需要一份完美的作品集，你需要一份「比別人多做一點」的作品集**。

**核心原則**：

1. **過程 > 成品**：HR 想看你怎麼想，不是你最後做出什麼。
2. **量化是王**：每個案例都有 1 個數字。
3. **故事性 > 個別案例**：整體要能串起一個成長軌跡。
4. **持續更新**：作品集不是做一次就結束，是長期累積的資產。
5. **TW 在地化**：CakeResume + LinkedIn 雙軌、案例選 TW 你真的常用的產品、通路選 Threads / Dcard / Medium 而不是只押 LinkedIn。

**行動建議**：到實習通查找你目標職位的真實心得（也可交叉參考 **Dcard 實習版、Yourator、CakeResume** 的職缺 JD 與面試回饋），看看 HR 在面試時會問什麼，挑出 3–4 個對應的案例放進你的作品集，立刻開始建構。今晚就花 2 小時建立第一個 Notion 頁面，這是你個人品牌的第一塊基石。
