# 數據視覺化：用 Looker Studio 把數字變故事

「請你做一份本月的行銷成效報告」——這句話翻譯成實習生聽得懂的版本，就是「把 Excel 裡的數字變成讓主管 5 秒看懂的圖」。但實習生最常犯的錯，是把「資料丟進工具」當作視覺化，最後做出一份 30 個圖表、3 種配色、看完還是不知道結論的儀表板。本篇教你如何用 Looker Studio（前身 Google Data Studio）把數字變成故事，並提供行銷、業務、內容三個職能最常做的儀表板模板。

## 一、為什麼選 Looker Studio？

### 1. 完全免費
Tableau、Power BI 都要付費（個人版每月 NT$ 300 起）。Looker Studio 是 Google 完全免費的工具，連結 Google Sheets、GA4、Google Ads、BigQuery 都是原生整合。

### 2. 無痛分享
產出的 dashboard 是一個網址，公司同事點開就能看，不需要安裝軟體。對實習生來說，「能讓主管在手機上看的儀表板」就是好儀表板。

### 3. 即時更新
資料來源更新，dashboard 自動刷新。你不需要每月手動 export 圖。**一次設好，每月省 4 小時**。

### 4. 學習曲線最低
比起 Tableau 需要兩三天學，Looker Studio 一個下午就能上手。**對實習生最友善**。

## 二、視覺化前的關鍵思考：什麼資料適合可視化？

實習生最大的誤區，是「看到數字就想做圖」。但**不是所有資料都適合可視化**。

### 適合可視化的資料
- 有「比較」的需求：A 渠道 vs B 渠道、本月 vs 上月
- 有「趨勢」的需求：每天、每週、每月的變化
- 有「比例」的需求：各類別在總體中的佔比
- 有「分布」的需求：客單價、年齡、停留時間的散佈
- 有「異常」需要凸顯：某天突然飆升或下降

### 不適合可視化的資料
- 單一數字：「本月營收 1,200 萬」直接寫一句話更清楚
- 表格資料：明細交易、客戶名單，用 table 比 chart 好
- 純文字結論：「本月最大問題是客服回應慢」

**先問自己：「這份資料想讓讀者比較什麼？」答不出來就不要做圖**。

## 三、四種圖表的選擇指南

實習生 90% 會用到的就是下面四種：

| 圖表類型 | 適合場景 | 不適合場景 |
|---------|---------|-----------|
| 長條圖 Bar | 比較不同類別的數量（渠道營收、產品銷量） | 連續時間趨勢 |
| 折線圖 Line | 時間趨勢（每日訪客、每月營收） | 類別比較 |
| 圓餅圖 Pie | 比例（少於 5 類別） | 超過 5 類、要比較變化 |
| 散佈圖 Scatter | 兩變量關係（廣告花費 vs 轉換數） | 單變量分析 |

### 幾個進階選擇
- **堆疊長條圖（Stacked Bar）**：同時看「總量」與「結構」，例如各月營收 + 各渠道貢獻。
- **Combo Chart（柱+線）**：左軸看「量」（如花費）、右軸看「率」（如 ROAS）。
- **熱圖（Heatmap）**：時段 × 星期幾的活躍度，一眼看出黃金時段。

### 圓餅圖的爭議
資料視覺化專家普遍認為「能不用圓餅就不用」，因為人眼不擅長比較角度。**超過 5 個類別時改用橫向長條圖**，視覺更清楚。

## 四、設計三大原則：少即是多

### 原則一：每個 dashboard 只回答一個核心問題
新手最常做出「什麼都有」的 dashboard，最後變成「什麼都看不到」。

**正確做法**：
- 給行銷主管的 dashboard：核心問題是「我們的廣告投資 ROI 如何？」
- 給 CEO 的 dashboard：核心問題是「公司本月健康嗎？」
- 給營運的 dashboard：核心問題是「哪些客戶即將流失？」

每個圖表都要服務這個核心問題，不相關的拿掉。

### 原則二：信息層次明確
讀者的眼睛該怎麼掃描你的 dashboard？
- 上方：最重要的 KPI（大數字 + 同比 / 環比）
- 中間：趨勢與分布（折線、長條）
- 下方：明細與例外清單（table）

**讀者花 5 秒看上方，5 分鐘看中間，5 分鐘看下方，三層各有用途**。

### 原則三：色彩克制
用一個主色 + 一個輔助色 + 灰色就夠。把「重要」的元素用主色凸顯，其他用灰色淡化。**新手常用 7 種顏色，結果一個都不重要**。

## 五、Looker Studio 操作速成

### 步驟一：連結資料源
登入 lookerstudio.google.com → Create → Data Source → 選擇來源（Google Sheets / GA4 / Google Ads / BigQuery / CSV upload）。

### 步驟二：建立 Report
Create → Report → 選你剛接好的 data source。畫面會打開一個空白 canvas。

### 步驟三：加圖表
頂部 toolbar → Add a chart → 選類型 → 拖到 canvas → 右側設定 Dimension（維度）與 Metric（指標）。

**核心觀念**：
- Dimension（維度）：分組欄位，文字類，例如「渠道」「日期」
- Metric（指標）：要被加總或平均的數值，例如「金額」「轉換數」

### 步驟四：加篩選（Filter Control）
頂部 → Add a control → 例如 Date range、Drop-down list。讓讀者能自己切時間或類別。

### 步驟五：分享
右上角 Share → 設權限（限指定人 / 公司網域 / 公開連結）。複製連結給主管。

## 六、與 Google Sheets / GA4 串接的實戰

### 串接 Google Sheets
最簡單的場景。你有一份「廣告每日花費」表在 Sheets，連到 Looker Studio。

**注意事項**：
- 第一列必須是「欄位名稱」
- 日期欄位請維持「日期」格式，不要混入「2026-04」這種字串
- 數字欄位不要混入「N/A」「-」，會被視為文字
- 如果 Sheets 結構會變，建議用「具名範圍」連接，避免之後找不到欄位

### 串接 GA4
GA4 的「探索」報表雖然強大，但難以分享。把 GA4 直接拉到 Looker Studio：

**最常見的指標組合**：
- 維度：頁面路徑、流量來源、裝置類別、國家
- 指標：使用者數、工作階段、跳出率、參與時間、轉換次數

**實習生情境**：行銷團隊每月要看「Top 10 流量來源帶來的轉換」，用 Looker Studio 一張長條圖搞定，每月自動更新。

### 串接 Google Ads
最直接的「廣告成效 dashboard」。指標：花費、曝光、點擊、CTR、CPC、轉換、CPA、ROAS。**這幾個是行銷實習生的「字母表」**，不熟記面試會被刷掉。

## 七、實習生最常做的三個 dashboard

### Dashboard 一：行銷成效週報

**核心問題**：本週每元廣告費賺回多少？

**版面建議**：
- 頂部 KPI：總花費、總營收、整體 ROAS、轉換數（本週 vs 上週）
- 中間趨勢：每日 ROAS 折線圖、每日花費 vs 營收柱+線
- 下方分組：各廣告平台 ROAS 長條圖、Top 10 廣告組明細

**目標讀者**：行銷主管、performance marketing 團隊

### Dashboard 二：業務 pipeline

**核心問題**：本季業績達成率多少？哪些 deal 卡住？

**版面建議**：
- 頂部 KPI：本季目標、已達成、達成率（圓環）、預計達成
- 中間：各業務員業績長條圖、各 stage deal 數量漏斗圖
- 下方：Top 20 deal 明細表、超過 30 天未推進的 deal 警示清單

**目標讀者**：業務主管、銷售總監

### Dashboard 三：內容績效（部落格 / IG / YouTube）

**核心問題**：哪些內容值得多投資？

**版面建議**：
- 頂部 KPI：本月發布數、總曝光、總互動、平均互動率
- 中間：每篇內容互動率長條圖、互動率 vs 觸及散佈圖
- 下方：Top 10 表現內容明細、Bottom 10（學什麼別做）

**目標讀者**：內容主管、社群編輯

## 八、新手最常踩的視覺化坑

### 坑一：圖太多、沒重點
一個 dashboard 塞 12 個圖。讀者看了 1 分鐘關掉。**砍到 5 個以內，每個都是必要**。

### 坑二：軸沒標清楚
「金額」是新台幣還是美金？「時間」是日還是週？沒標的話，讀者腦中會自己腦補錯誤。**每個軸都要清楚 label + 單位**。

### 坑三：百分比與絕對值搞混
「轉換率成長 5%」是從 10% 變 15%（漲 5 個百分點）還是從 10% 變 10.5%（漲 5%）？這個誤會會讓報告被退件。**寫清楚是「百分點」還是「百分比」**。

### 坑四：用截圖當 dashboard
有些實習生把 Looker Studio 結果截圖貼到簡報。錯！直接 share 連結，主管自己點開能切時間區間。**互動性是 dashboard 的核心價值**。

### 坑五：沒有「結論文字」
一份只有圖表的 dashboard，等於把分析的工作丟回給主管。**頂端應該寫一段「本週重點 3 句話」**，例如：「本週 ROAS 從 4.2 降到 3.5，主因是 Meta 受眾疲乏，建議下週測試新素材。」

## 九、FAQ：實習生問視覺化常見問題

**Q1：Looker Studio 跟 Tableau 哪個該學？**
實習階段先學 Looker Studio。免費、上手快、跟 Google 生態整合。**進職場後若進到資料部門，再補 Tableau**。

**Q2：Excel 樞紐分析做的圖能直接給主管看嗎？**
可以，但限於「一次性報告」。**重複性的週報、月報，請用 Looker Studio 做成 dashboard**，否則你會花一輩子在重做圖。

**Q3：dashboard 要做多漂亮？**
不必。實習階段「資訊正確、結構清楚、結論明確」三件事比配色重要 100 倍。**看起來像極簡的 dashboard，比花俏的更顯專業**。

**Q4：要學會自訂計算欄位（Calculated Field）嗎？**
要。這是把 Looker Studio 從「拼圖工具」升級成「分析工具」的關鍵。常用的：

```
ROAS = SUM(Revenue) / SUM(Cost)
轉換率 = SUM(Conversions) / SUM(Sessions)
本月 vs 上月變化 = (本月 - 上月) / 上月
```

**Q5：dashboard 多久更新一次？**
看資料源。如果連 Google Ads 即時，dashboard 預設 12 小時更新；連 Sheets 是 15 分鐘。**「即時」常常是過度需求**，多數情境一天一次就夠。

## 十、結語：好的視覺化，是讓主管 5 秒做決策

實習生最大的成長機會，不是「做出漂亮的圖」，而是「做出能影響決策的圖」。當你的 dashboard 讓主管在會議裡 5 秒就拍板「下週把 Meta 預算砍 30%、轉到 Google」，**你就從工讀生變成了參與決策的人**。

**三個關鍵原則**：
1. **每個 dashboard 只回答一個核心問題**：什麼都想顯示，等於什麼都沒顯示。
2. **資訊層次比配色重要**：頂部 KPI、中間趨勢、下方明細，三層讓讀者各取所需。
3. **圖表要附結論文字**：「給主管自己解讀」是失職，「我已經看完並下了結論」才是價值。

**行動建議**：到實習通逛行銷分析、PM、業務分析類的實習心得，留意「他們交了什麼樣的 dashboard」「主管是否拿來開會」。今天就用 Google Sheets 隨便做一份模擬資料，連到 Looker Studio，做完上面三個 dashboard 中的一個，下週你的履歷就能多一行真本事。
